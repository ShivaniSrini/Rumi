### Definition of Done:
This document defines all the features and user stories in the Rumi Habit tracker are considered complete. According to the sprint 0, the definition focuses on planning, documentation and designing user interfaces must be complete for future development.

### 1. Feature planning and completion criteria
Features are considered done  for sprint 0 when it is completely described with:
- Specific functionality that is clearly described and aligns with the project's main purpose
- All features are intended to have a positive impact on user experience
### 2. Documentation
Every user story or feature must include clear documentation:
      - The purpose of the feature and how it is supposed to work
      - Rules or behaviours of the feature towards the project
      
### 3. User Interfaces and User Experience Planning
- Wireframes must be completed and reviewed by the team
- Focuses on clarity, and all MVP features must be visible

### 4 Project's unique features 
All the following unique features must be presented in the future UI:
- Gentle reminders
- Supportive messages 
- Rewards to boost the positive dopamine hit for task completions and regular logins 
- Mood analytics and habit progress reports.

### Conclusion
Based on the sprint 0, the definition of done for our team is fully planned, documented and agreed upon for all the requirements listed above.
